# AGENTS.md

## 协作规则

1. 函数需要有注释，风格是 Google，使用中文。
2. 用户提出写测试时，除非明确要求，否则禁止使用命令行调用形式，方便用户在 IDE 中运行和 debug。
3. 生成代码必须简洁高效，禁止过度设计和不必要冗余，需要便于阅读理解，代码不能写得太长，需要解耦并分模块开发。
4. Agent 自己写的测试函数可以使用 pytest，并可在命令行中运行，运行代码时默认使用 `uv run`。
5. 生成脚本代码必须是函数式、可在 IDE 中运行，注意路径问题，禁止生成只能直接在命令行运行的脚本。
6. 运行过程中的功能改动和逻辑需要及时更新到本文件，功能改动及时提交 git，并使用 `git-commit-conventions` skill。
7. 生成的函数禁止使用 argparse，全部使用函数化调用，输出参数和结果，便于 IDE 调试。
8. 生成的功能性代码需要有 `if __name__ == "__main__":`，并写出函数和对应使用注释，方便使用和调试，输入必须参数化。
9. 默认使用 SQLAlchemy 操作 SQL。
10. 需要自动判断开发方式，简单任务使用 inline 模式，复杂任务再启用 subagent 或 multiagent。
